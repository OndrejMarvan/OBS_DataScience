What we do today (90 minutes)

  • sf basics: points → lines → polygons
  • Reading spatial data (st_read()), checking CRS
  • 2–3 core spatial operations: transform, buffer, within/intersects
  • Minimal mapping with geom_sf()
  • Legacy bridge: sf ↔︎ sp conversions

Mental model: sf is tidy data + geometry

  • Spatial data = attributes (variables) + geometry (where/shape)
  • sf behaves like a data.frame/tibble, with one special column: geometry

Key classes:

  • sf — feature collection (table)
  • sfc — geometry column (list-column)
  • sfg — single geometry (POINT / LINESTRING / POLYGON)

Before we start: pipes %>% (quick intuition)

Pipes let you write transformations as a readable sequence:

# Equivalent notations:
# function3(function2(function1(object)))
# object %>% function1() %>% function2() %>% function3()

Why we like them here:

  • sf objects are tidy-ready
  • dplyr verbs (filter/mutate/…) work naturally on sf
