    Python and SQL: intro / SQL platforms

    How to pass this class?

    A. Final Test  (50%)

    • Format: Multiple-choice questions (MCQs) (A multiple-choice
      question is a type of objective assessment in which a question has
      zero or more possible answers)
    • Number of questions: 25
    • Time limit: 90 minutes
    • Date of exam – during the examination session: 26 January 2026 –
      8 February 2026 (to be determined later by the administration
      office)

    B. Project Presentation (50%)

    • Project must be done individually 
    • Deadlines for project presentation and code submission is 7.01.2026 (if file too big to submit via moodle then submit link to google drive)
    • you have 10 minutes to present

    What should be in the final project:

    Define a Problem or Dataset of Interest

      • Choose a dataset (from Kaggle, open data portals, or self-collected).

      • Formulate a clear research question or goal (e.g., predicting trends, analyzing patterns, building a small tool).

    • Data Handling & Wrangling (Pandas + NumPy)

      • Import and clean raw data (handle missing values, duplicates, inconsistent types).

      • Perform exploratory data analysis (summaries, aggregations, transformations).

    • Apply Logic & Functions

      • Use Python functions and control flow (if-statements, loops) to structure their analysis.

      • Possibly implement simple algorithms (like calculating similarity scores, classification rules).

    • Visualization (Seaborn, Matplotlib)

      • Create clear, story-driven plots (trends, comparisons, distributions).

      • Include at least 5–10 types of plots that highlight insights.

    • SQL Integration

      • Store or query data from a database.

      • Demonstrate joins, filtering, or aggregations.

    • Presentation Layer (Streamlit or Django)

      • Build a simple dashboard or mini web app to showcase results.

      • For Streamlit: interactive charts, filters, and tables.

      • For Django: a structured web app

    • In submission you need to to send via moodle code and presentation in pdf
